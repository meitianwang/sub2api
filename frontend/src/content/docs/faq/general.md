# 通用问题

## Q: 接口地址（Base URL）是什么？

在控制台「API Keys」页面顶部可以查看接口地址。如果平台提供了多个线路，可以点击测速按钮，选择延迟最低的线路以获得最佳体验。

## Q: 一个 API Key 可以调用多个模型吗？

可以，但仅限于该 Key 所属分组中包含的模型。如果你需要使用不同分组中的模型，需要分别创建绑定到对应分组的 API Key。

例如，如果分组 A 包含 Claude 系列模型，分组 B 包含 GPT 系列模型，那么你需要两个不同的 Key 分别调用这两个分组的模型。

## Q: 支持流式输出吗？

支持。根据不同的 API 格式，设置方式如下：

- **OpenAI 格式**：在请求体中设置 `"stream": true`
- **Anthropic 格式**：在请求体中设置 `"stream": true`
- **Gemini 格式**：使用 `streamGenerateContent` 端点替代 `generateContent`

## Q: 余额会过期吗？

充值余额不会过期，长期有效。你可以根据自己的使用节奏按需充值，无需担心余额失效。

## Q: 如何查看 API 调用记录？

登录控制台，在「活动日志」页面可以查看每次请求的详细信息，包括：

- 调用的模型名称
- Token 用量（输入/输出）
- 请求延迟
- 本次请求费用
- 请求状态（成功或失败）

支持按时间范围筛选，方便追踪用量和排查问题。

## Q: API Key 可以设置额度上限吗？

可以。在创建或编辑 API Key 时，可以设置该 Key 的额度限制。当累计消费达到设定的上限后，该 Key 将自动停用，后续请求将被拒绝。这一功能可以有效防止意外超支。

如需恢复使用，可以在控制台中调高额度上限或重新启用该 Key。

## Q: 提示 401 Unauthorized 怎么办？

遇到 401 错误时，请按以下步骤排查：

1. **检查 API Key 是否正确** -- 确认请求头中的 Key 与控制台中显示的一致，注意不要包含多余的空格或换行符
2. **确认 Key 状态** -- 登录控制台，检查该 Key 的状态是否为「活跃」。如果 Key 已被停用或额度已用尽，需要重新启用或调整额度
3. **检查 IP 白名单** -- 如果创建 Key 时设置了 IP 白名单，确认当前发起请求的 IP 地址在白名单范围内

## Q: 提示模型不可用怎么办？

遇到模型不可用的错误时，请检查以下几点：

1. **确认分组支持该模型** -- 登录控制台查看当前 API Key 所属分组包含的模型列表，确认目标模型在列表中
2. **检查模型名称拼写** -- 在「模型」页面确认模型的准确名称，注意大小写和版本号。例如 `claude-sonnet-4-6` 不能写成 `claude-sonnet4`

## Q: 如何联系客服/获取帮助？

如有问题无法通过文档自行解决，请通过平台提供的联系方式联系管理员。反馈问题时，建议提供以下信息以便快速定位：

- 使用的 API Key 前缀（如 `sk-abcd...`，请勿提供完整 Key）
- 请求的模型名称
- 完整的错误信息或错误码
- 请求发生的大致时间
