# 分组介绍

## 什么是分组

AIInterface 平台将模型按照上游供应商/渠道划分为不同的**分组（Group）**。每个分组对应一个或多个上游通道，决定了该分组下可用的模型列表、定价倍率以及支持的 API 格式。

API Key 在创建时需要绑定到一个特定的分组。绑定后，该 Key 只能调用所属分组中包含的模型，并按照该分组的定价进行计费。

## 分组之间的差异

不同分组在以下方面可能存在差异：

### 支持的模型

每个分组包含的模型列表不同。例如：

- 分组 A 可能包含 Claude 系列模型（claude-sonnet-4-20250514、claude-opus-4-20250514 等）
- 分组 B 可能包含 GPT 系列模型（gpt-4o、o3 等）
- 分组 C 可能包含 Gemini 系列模型（gemini-2.5-pro 等）
- 也可能存在包含多个供应商模型的综合分组

### 定价倍率

每个分组有独立的定价倍率，决定了实际费用与官方定价的比例关系：

- 倍率 1.0 = 与官方定价一致
- 倍率 0.5 = 官方定价的 50%（半价）
- 倍率 0.1 = 官方定价的 10%（一折）

倍率越低，使用成本越低。

### 平台类型

分组的平台类型决定了该分组支持的 API 请求格式：

- **Anthropic 平台** -- 使用 Anthropic Messages API 格式
- **OpenAI 平台** -- 使用 OpenAI Chat Completions API 格式
- **Gemini 平台** -- 使用 Google Gemini API 格式

使用时需要确保你的客户端发送的请求格式与分组的平台类型匹配。

### 计费类型

- **按量计费（标准）**：按实际使用的 Token 数量计费，从账户余额中扣除
- **订阅制**：按订阅周期付费，在订阅有效期内不限量使用

## 如何选择分组

选择分组时，建议从以下几个维度考虑：

### 1. 确认使用场景和工具

不同的 CLI 工具和应用对 API 格式有特定要求：

- **Claude Code** -- 需要 Anthropic 平台类型的分组
- **Codex CLI** -- 需要 OpenAI 平台类型的分组
- **Gemini CLI** -- 需要 Gemini 平台类型的分组
- **其他兼容 OpenAI 格式的工具** -- 选择 OpenAI 平台类型的分组

### 2. 查看定价倍率

在预算有限的情况下，优先选择倍率较低的分组，以降低使用成本。

### 3. 确认所需模型

确认目标分组包含你需要使用的模型。例如，如果你需要使用 claude-sonnet-4-20250514，则需要选择包含该模型的分组。

## 在哪里查看分组信息

- **创建 API Key 时**：在控制台创建 API Key 的页面，可以看到所有可选分组及其详细信息
- **模型页面**：访问导航栏中的「模型」页面，可以查看所有公开分组及其包含的模型和定价

## 多分组使用

你可以创建多个 API Key，分别绑定到不同的分组，以满足不同的使用需求。例如：

- 创建一个绑定 Anthropic 分组的 Key，用于 Claude Code
- 创建一个绑定 OpenAI 分组的 Key，用于 Codex CLI
- 创建一个绑定低倍率分组的 Key，用于日常测试

每个 Key 独立计费，互不影响。
