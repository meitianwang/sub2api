# 分组介绍

## 什么是分组

AIInterface 平台将模型按照渠道和用途划分为不同的**分组（Group）**。每个分组决定了可用的模型列表、计费价格以及支持的 API 格式。

API Key 在创建时需要绑定到一个特定的分组。绑定后，该 Key 只能调用所属分组中包含的模型，并按照该分组的定价进行计费。

## 分组之间的差异

不同分组在以下方面可能存在差异：

### 支持的模型

每个分组包含的模型列表不同。例如：

- 分组 A 可能包含 Claude 系列模型（claude-sonnet-4-6、claude-opus-4-6 等）
- 分组 B 可能包含 Gemini 系列模型（gemini-2.5-pro 等）
- 也可能存在同时包含多个系列模型的综合分组

### 计费价格

不同分组对同一模型的计费价格可能不同。具体价格可在[「模型」]({{BASE_URL}}/models)页面查看各分组的详细定价。

### 支持的 API 格式

分组支持的 API 格式取决于其包含的模型系列：

- 包含 Claude 模型的分组支持 OpenAI 兼容格式和 Anthropic 原生格式
- 包含 Gemini 模型的分组支持 Gemini 原生格式

## 如何选择分组

选择分组时，建议从以下几个维度考虑：

### 1. 确认所需模型

确认目标分组包含你需要使用的模型。例如，如果你需要使用 claude-sonnet-4-6，则需要选择包含该模型的分组。

### 2. 比较计费价格

在预算有限的情况下，可以比较不同分组对同一模型的定价，选择价格合适的分组。

## 在哪里查看分组信息

- **创建 API Key 时**：在控制台创建 API Key 的页面，可以看到所有可选分组及其详细信息
- **模型页面**：访问导航栏中的[「模型」]({{BASE_URL}}/models)页面，可以查看所有公开分组及其包含的模型和定价

## 多分组使用

你可以创建多个 API Key，分别绑定到不同的分组，以满足不同的使用需求。例如：

- 创建一个绑定高质量分组的 Key，用于 Claude Code / Codex CLI
- 创建一个绑定低价格分组的 Key，用于日常测试

每个 Key 独立计费，互不影响。
