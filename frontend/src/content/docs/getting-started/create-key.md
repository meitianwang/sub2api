# 创建 API Key

API Key 是调用平台接口的凭证。你需要至少创建一个 Key 才能开始使用 API 服务。

## 创建步骤

1. 登录控制台，进入[「API Keys」]({{BASE_URL}}/keys)页面
2. 点击「创建新 Key」按钮
3. 填写以下信息：
   - **名称** -- 为 Key 取一个便于识别的名称，例如「Claude Code 开发」
   - **分组** -- **务必正确选择分组**。分组决定了这个 Key 能调用哪些模型以及对应的计费价格，选错分组可能导致无法调用目标模型或费率不符合预期
   - **过期时间**（可选）-- 设置 Key 的有效期，留空表示永不过期
   - **额度限制**（可选）-- 设置此 Key 的最大可用额度，用于控制成本
   - **模型限制**（可选）-- 限制 Key 只能调用指定模型
   - **IP 限制**（可选）-- 白名单：限制 Key 只能从指定 IP 地址发起请求；黑名单：阻止指定 IP 地址使用该 Key
4. 确认无误后点击提交

## 复制并保存 Key

Key 创建成功后会**完整显示一次**，请立即复制并妥善保存。关闭弹窗后将无法再次查看完整 Key。

> 如果遗失了 Key，只能删除旧 Key 并重新创建。

## 使用建议

- 每个 Key 绑定一个分组，**无法在创建后更改分组**
- 建议为不同用途创建独立的 Key，例如开发测试和生产环境分别使用不同 Key
- 在 Key 列表中，点击「使用密钥」按钮可以查看针对各种 CLI 工具（如 Claude Code、Gemini CLI、Codex CLI 等）的现成配置片段，直接复制即可使用
- 定期检查 Key 的用量，及时清理不再使用的 Key
